# Career OS — Ngoc Le

A working repository for running a senior product career like a product: a clear
strategy, sharp positioning assets, defensible evidence, and an actively managed
opportunity pipeline.

The folders are numbered as a workflow, not just a filing system — read top to
bottom the way you'd actually run a search: decide direction → sharpen what you
sell → back it with proof → run the hunt → reuse tooling → keep growing.

## Map

| Folder | Job it does | What lives here |
| :----- | :---------- | :-------------- |
| `00-strategy/` | **Career management** — direction & decisions | Positioning thesis, goals/targets, and the `career-radar` market-decision tool |
| `01-assets/` | **The product** — positioning collateral | Résumé (master + tailored cuts), portfolio, selected work, competence knowledge base |
| `02-evidence/` | **Proof library** — raw material behind claims | Trusting Social milestones, guardrail architecture, and the state-machine config |
| `03-pipeline/` | **Opportunity hunting** — execution | `tracker.md` (opportunity CRM) + one self-contained folder per target company |
| `04-prep-toolkit/` | **Reusable across every opportunity** | Interview roleplay builder, question banks |
| `05-growth/` | **Learning & thought leadership** | Reading lists and research |

## Operating principles

- **One master résumé, many tailored cuts.** The canonical résumé lives in
  `01-assets/resume/master/`. Every company-specific version is a derivative in
  `tailored/` — never edit two masters.
- **Assets cite evidence; they don't restate it.** When a claim needs proof (the
  17-state compliance architecture, the 5.2x revenue uplift), the source of truth
  is in `02-evidence/`. Keep the receipts separate from the pitch.
- **Each opportunity is self-contained.** Everything about a target company lives
  under `03-pipeline/targets/<company>/`. To pursue a new one, copy the folder
  pattern and add a row to `tracker.md`.
- **Promote anything reusable.** If a prep artifact would help at the *next*
  interview too, it belongs in `04-prep-toolkit/`, not buried in one target.
